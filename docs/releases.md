# Releasing UltraX

## Versioning

Use patch releases for focused improvements:

```powershell
npm version 1.0.6 --no-git-tag-version
```

Update `CHANGELOG.md` before tagging.

## Local Release Build

```powershell
npm ci
npm run typecheck
npm run build
npm run dist:win
```

Expected local artifacts:

- `release/UltraX Setup <version> x64.exe`
- `release/UltraX <version> Portable x64.exe`
- update metadata files generated by electron-builder when publishing

Do not commit `release/`.

## GitHub Repository Setup

Create a GitHub repository manually, then run:

```powershell
git init
git add .
git commit -m "Initial UltraX Browser release"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/ultrax-browser.git
git push -u origin main
```

Replace `YOUR_USERNAME/ultrax-browser` with the real repository.

## GitHub Actions Release

The release workflow runs on tags:

```powershell
git tag v1.0.6
git push origin v1.0.6
```

The workflow:

- installs dependencies with `npm ci`
- typechecks
- builds
- packages Windows NSIS and Portable builds
- publishes artifacts to GitHub Releases through `electron-builder --publish always`

## Required Secrets

The workflow can publish with the built-in `GITHUB_TOKEN`.

For production signing, add:

- `WIN_CSC_LINK`
- `WIN_CSC_KEY_PASSWORD`

Do not commit signing certificates or passwords.

## Auto-Updater Metadata

`electron-builder` publishes update metadata for `electron-updater` when GitHub Releases are configured and `--publish always` is used.

## Production Checklist

- Replace placeholder repository links.
- Protect `main`.
- Require the build workflow.
- Sign release tags.
- Configure Windows code signing.
- Verify installer and portable EXE on a clean Windows machine.
